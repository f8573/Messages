ROLLBACK PROCEDURE
===============================================================================

IF ISSUES OCCUR:

1. Apply down migration:
   psql postgresql://dev:dev@localhost:5432/dev < \
     migrations/000043_search_quality_improvements.down.sql

2. Remove tracking:
   psql postgresql://dev:dev@localhost:5432/dev -c \
     "DELETE FROM schema_migrations WHERE filename = '000043_search_quality_improvements.up.sql';"

3. Restart application
   - Old search behavior restored
   - No data loss except new_columns (acceptable)
   - Application continues working

TIME TO ROLLBACK: ~1-2 minutes

===============================================================================
SUPPORT & TROUBLESHOOTING
===============================================================================

COMMON ISSUES:

1. "Extension unaccent not available"
   - Install PostgreSQL contrib: apt-get install postgresql-contrib
   - Or rebuild with contrib support

2. "Migration takes too long"
   - Normal for large databases (many messages)
   - Monitor with: SELECT COUNT(*), COUNT(search_vector_en) FROM messages;
   - Can take 5-10 minutes for millions of messages

3. "Permission denied creating extension"
   - User needs superuser or CREATE EXTENSION privilege
   - Run: ALTER USER dev SUPERUSER;

4. "Indices not being used in queries"
   - Check with EXPLAIN ANALYZE
   - May need ANALYZE table to update statistics
   - Run: ANALYZE messages;

5. "Search returns no results"
   - Check query normalization behavior
   - Try: q=test (at least 2 characters)
   - Use fuzzy mode for typos: search_mode=fuzzy

For more help, see MIGRATION_GUIDE.md "Troubleshooting" section
