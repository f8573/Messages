# P2.3 Implementation — Visual Flow & Architecture

## Release Lifecycle with Immutability

```
┌─────────────────────────────────────────────────────────────────────────┐
│ PHASE 1: RELEASE CREATION                                              │
└─────────────────────────────────────────────────────────────────────────┘

    POST /v1/apps/register
           │
           ▼
    ┌──────────────────────┐
    │ Decode Manifest      │
    │ Validate Structure   │
    └──────────────────────┘
           │
           ▼
    ┌──────────────────────────────────────────┐
    │ Compute ManifestHash (old, not changed)  │ (SHA-256)
    │ Compute ManifestContentHash (NEW)        │ (SHA-256 of JSON)
    │                                          │
    │ Store both:                              │
    │ - manifest_hash: for version uniqueness  │
    │ - manifest_content_hash: for immutability│
    └──────────────────────────────────────────┘
           │
           ▼
    ┌──────────────────────────────────────────┐
    │ Create Release Record                    │
    │ Status: "draft" (for dev releases)       │
    │ or "submitted" (for prod releases)       │
    └──────────────────────────────────────────┘
           │
           ▼
    ┌──────────────────────────────────────────┐
    │ Save to Database                         │
    │ - manifest_json: blob                    │
    │ - manifest_hash: version identifier      │
    │ - manifest_content_hash: immutability key│ ◄─── NEW
    │ - asset_set_hash: NULL (not yet set)     │ ◄─── NEW
    │ - immutable_at: NULL (not yet set)       │ ◄─── NEW
    └──────────────────────────────────────────┘
           │
           ▼
    Response to client:
    {
      "version": "1.0.0",
      "review_status": "draft",
      "manifest_content_hash": "sha256:abc123...",  ◄─── NEW
      "asset_set_hash": null,                        ◄─── NEW
      "immutable_at": null                           ◄─── NEW
    }


┌─────────────────────────────────────────────────────────────────────────┐
│ PHASE 2: ADMIN APPROVAL (IMMUTABILITY ENFORCEMENT)                     │
└─────────────────────────────────────────────────────────────────────────┘

    POST /v1/apps/{id}/{version}/approve (admin)
           │
           ▼
    ┌──────────────────────────────────────────┐
    │ Load Release from Database               │
    │ - manifest_json (stored blob)            │
    │ - manifest_content_hash (stored at creation)
    └──────────────────────────────────────────┘
           │
           ▼
    ┌──────────────────────────────────────────┐
    │ IMMUTABILITY VALIDATION                  │ ◄─── NEW ENFORCEMENT
    │                                          │
    │ Current = recompute SHA-256(manifest)    │
    │ Stored  = load manifest_content_hash     │
    │                                          │
    │ if Current == Stored:                    │
    │    ✓ Manifest unchanged                  │
    │    ✓ No tampering detected               │
    │    → PROCEED to asset hash computation   │
    │ else:                                    │
    │    ✗ REJECT with error                   │
    │    ✗ "manifest_has_changed..."          │
    │    → ABORT (approval fails)              │
    └──────────────────────────────────────────┘
           │
           ├─── ERROR PATH ──► Return 400 Bad Request
           │                 Code: "immutability_violation"
           │
           ▼ (SUCCESS PATH)
    ┌──────────────────────────────────────────┐
    │ Compute Asset Set Hash                   │ ◄─── NEW
    │                                          │
    │ assetHashes = [                          │
    │   manifest_content_hash,                 │
    │   entrypoint_origin,                     │
    │   preview_origin                         │
    │ ]                                        │
    │ assetHashes.Sort()                       │
    │ asset_set_hash = SHA-256(concat(hashes)) │
    └──────────────────────────────────────────┘
           │
           ▼
    ┌──────────────────────────────────────────┐
    │ Mark Release as Immutable                │ ◄─── NEW
    │                                          │
    │ release.ReviewStatus = "approved"        │
    │ release.ReviewedAt = now()               │
    │ release.PublishedAt = now()              │
    │ release.ImmutableAt = now()              │ ◄─── NEW: IMMUTABILITY SET
    │ release.AssetSetHash = computed_hash     │ ◄─── NEW: INTEGRITY BOUND
    └──────────────────────────────────────────┘
           │
           ▼
    ┌──────────────────────────────────────────┐
    │ Save to Database                         │
    │ - All hash fields now populated          │
    │ - immutable_at timestamp recorded        │
    │ - Asset references can be logged         │
    └──────────────────────────────────────────┘
           │
           ▼
    ┌──────────────────────────────────────────┐
    │ Log Audit Entry                          │
    │ - action: "release.transition"           │
    │ - next_status: "approved"                │
    │ - metadata: {hash: "sha256:...", ...}    │
    └──────────────────────────────────────────┘
           │
           ▼
    Response to admin:
    {
      "version": "1.0.0",
      "review_status": "approved",
      "manifest_content_hash": "sha256:abc123...",  ◄─── SAME (unchanged)
      "asset_set_hash": "sha256:def456...",          ◄─── NOW SET
      "immutable_at": "2026-03-21T12:34:56Z"         ◄─── NOW SET
    }


┌─────────────────────────────────────────────────────────────────────────┐
│ PHASE 3: USAGE (IMMUTABILITY GUARANTEED)                               │
└─────────────────────────────────────────────────────────────────────────┘

    GET /v1/apps/{id}
           │
           ▼
    ┌──────────────────────────────────────────┐
    │ Load Release                             │
    │ - All hashes now populated               │
    │ - immutable_at shows when it became      │
    │   permanently immutable                  │
    └──────────────────────────────────────────┘
           │
           ▼
    Response to client:
    {
      "releases": [
        {
          "version": "1.0.0",
          "review_status": "approved",
          "manifest_content_hash": "sha256:abc123...",  ◄─── For verification
          "asset_set_hash": "sha256:def456...",          ◄─── For integrity
          "immutable_at": "2026-03-21T12:34:56Z"         ◄─── When immutable
        }
      ]
    }

    Client can verify:
    1. Compute SHA-256(manifest) locally
    2. Compare with manifest_content_hash from response
    3. If match: Content verified as unchanged ✓


┌─────────────────────────────────────────────────────────────────────────┐
│ FUTURE PHASE: CONTENT-ADDRESSED STORAGE                                │
└─────────────────────────────────────────────────────────────────────────┘

    Current: /cdn/miniapps/myapp/manifest.json
    Future:  /cdn/miniapps/myapp/1.0.0-sha256:abc123/manifest.json
                                           └────────────────────┘
                                           Hash in URL = immutability
                                           CDN caches forever


┌─────────────────────────────────────────────────────────────────────────┐
│ ERROR SCENARIOS & DETECTION                                            │
└─────────────────────────────────────────────────────────────────────────┘

Scenario 1: Tampering After Creation
─────────────────────────────────────
    1. Release created:
       manifest_content_hash = SHA-256(original_manifest) = "abc123"

    2. Attacker modifies manifest_json in database (if compromised)
       manifest_json = TAMPERED_CODE

    3. Admin approves:
       VALIDATION:
       current_hash = SHA-256(manifest_json) = SHA-256(TAMPERED) = "xyz789"
       stored_hash  = "abc123"

       IF current_hash != stored_hash:
           REJECT ✗ with error: "manifest_has_changed_after_creation"

    4. Approval fails → tampering detected and prevented


Scenario 2: Hash Collision Attempt
──────────────────────────────────
    Attacker tries to find different content with same hash:

    This is cryptographically impossible:
    - SHA-256 = 256 bits = 2^256 possible values
    - Probability of collision = 2^-128 (practically zero)
    - Same as breaking entire security infrastructure

    Mitigation: Use SHA-256 (already standard in crypto)


Scenario 3: Concurrent Approvals
────────────────────────────────
    Multiple admins approve same release simultaneously:

    Database: pg_advisory_xact_lock(registryAdvisoryLockID)

    Result: Serialized (one approval succeeds, others see already-approved)
    Both show same hashes = idempotent


Scenario 4: Legacy Releases (Pre-2026-03-21)
──────────────────────────────────────────
    Old releases may have NULL hash fields:

    Validation in validateManifestImmutability():
    if manifest_content_hash == "":
        return nil  // Skip validation for legacy releases

    Result: Legacy releases can still be approved (backward compatible)


┌─────────────────────────────────────────────────────────────────────────┐
│ DATA FLOW: Hash Computation                                            │
└─────────────────────────────────────────────────────────────────────────┘

At Creation:
─────────
    manifest map
         │
         ▼
    json.Marshal()
         │
         ▼
    manifestBytes = {
      "app_id": "myapp",
      "version": "1.0.0",
      ...
    }
         │
         ▼
    SHA-256(manifestBytes)
         │
         ▼
    hex encode
         │
         ▼
    fmt.Sprintf("sha256:%x", hash)
         │
         ▼
    "sha256:abc123def456..."  ← Stored in DB


At Approval:
──────────
    Stored JSON blob
         │
         ▼
    json.Marshal() again
         │
         ▼
    SHA-256(manifestBytes)  [recomputed]
         │
         ▼
    Compare with stored manifest_content_hash
         │
         ├─ MATCH → ✓ Continue to approval
         │
         └─ MISMATCH → ✗ Reject with error


Asset Set Hash Computation:
──────────────────────────
    Assets = [
      manifest_content_hash,
      entrypoint_origin,
      preview_origin
    ]
         │
         ▼
    Sort deterministically
         │
         ▼
    Concatenate: "hash1hash2hash3"
         │
         ▼
    SHA-256() of concatenated string
         │
         ▼
    "sha256:xyz789..."  ← Stored at approval


┌─────────────────────────────────────────────────────────────────────────┐
│ KEY INVARIANTS                                                         │
└─────────────────────────────────────────────────────────────────────────┘

1. MANIFEST IMMUTABILITY
   ┌─────────────────────────────────────────┐
   │ Once created, manifest_content_hash     │
   │ is never modified (immutable baseline)  │
   └─────────────────────────────────────────┘

2. HASH VALIDATION AT APPROVAL
   ┌─────────────────────────────────────────┐
   │ Before approval, hash re-computed from  │
   │ stored JSON and compared. Mismatch =    │
   │ rejection (tampering detected)          │
   └─────────────────────────────────────────┘

3. IMMUTABLE TIMESTAMP
   ┌─────────────────────────────────────────┐
   │ immutable_at set only once at approval. │
   │ Marks when release became permanently   │
   │ immutable.                              │
   └─────────────────────────────────────────┘

4. ASSET SET BINDING
   ┌─────────────────────────────────────────┐
   │ asset_set_hash computed from all        │
   │ related assets. Commitment to complete  │
   │ asset set, not just manifest.           │
   └─────────────────────────────────────────┘

5. BACKWARD COMPATIBILITY
   ┌─────────────────────────────────────────┐
   │ Legacy releases without hashes still    │
   │ work. Validation skipped if hash NULL.  │
   │ New releases get hashes automatically.  │
   └─────────────────────────────────────────┘


┌─────────────────────────────────────────────────────────────────────────┐
│ SECURITY PROOF: IMMUTABILITY                                           │
└─────────────────────────────────────────────────────────────────────────┘

Claim: "Once approved, release is immutable"

Proof:
──────
1. Release created with manifest_content_hash = H₀

2. At approval, hash recomputed = H_current
   If H_current ≠ H₀, approval REJECTED
   Else, release marked immutable (immutable_at set)

3. After approval (future access):
   If manifest blob modified in storage:
      - Hash recomputed = H_modified
      - H_modified ≠ H₀ (unless collision, which is cryptographically impossible)
      - Verification fails → tampering detected

4. Once immutable_at is set:
   - Release record is read-only (application-level enforcement)
   - New versions required for any changes
   - Old version remains with original hash (immutable)

Therefore: Release is immutable once approved ✓
