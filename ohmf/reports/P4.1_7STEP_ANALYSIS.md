## STEP 1: UNDERSTAND

### What This Solves

**Immutable Audit Trail**
- Every session lifecycle event is recorded permanently
- No modification or deletion allowed (database-level enforcement)
- Enables compliance auditing, forensics, and dispute resolution

**Session Reconstruction**
- From event log alone, can rebuild session state at any point in time
- Events include actor_id, timestamp, and contextual metadata
- Enables "replay" capability for debugging or recovery

**Semantic Milestones**
- 5 event types mark distinct phases: creation → joins → storage updates → snapshots → message projections
- Each event captures WHO (actor_user_id), WHAT (event_type), WHEN (created_at), and WHY (metadata/body)

**Risk Without It**
- **No Replay:** Can't reconstruct session from immutable events
- **No Audit Trail:** No proof of state changes, actors, or timestamps
- **Compliance Gaps:** No immutable records for regulatory requirements (SOC2, HIPAA)
- **Forensics Blind:** Can't debug multi-user sessions without event history

### Why Events Are Semantic Milestones

1. **session_created** — Session initialized with participants & permissions
2. **session_joined** — New participant enters session
3. **storage_updated** — App calls bridge method (implicit, stored in AppendEvent)
4. **snapshot_written** — State snapshot persisted with version increment
5. **message_projected** — Messages synchronized to session (future)

Each milestone represents a **distinct state transition** that should be recorded:
- **Immutable:** Once written, cannot be changed
- **Atomic:** All or nothing (transactional for snapshots)
- **Timestamped:** exact timing of state change
- **Auditable:** actor_id tracks WHO made the change

### What Events Define
```
Session → participants → permissions → state transitions → events

            session_created (event_seq=1)
                    ↓
            session_joined (event_seq=2) — another participant
                    ↓
            storage_updated (event_seq=3) — bridge call
                    ↓
            snapshot_written (event_seq=4) — state persisted
```

- **1 query function:** GetSessionEvents() with filtering and pagination

CREATE TRIGGER miniapp_events_append_only_trigger
BEFORE UPDATE OR DELETE ON miniapp_events
FOR EACH ROW
EXECUTE FUNCTION enforce_miniapp_events_append_only();
