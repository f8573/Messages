**Current Implementation:** Mini-app runtime (miniapp-runtime.js) already uses Bearer tokens correctly:

```javascript
headers.set("Authorization", `Bearer ${state.auth.accessToken}`);
```

- ✅ Token sent in Authorization header (not cookies)
- ✅ No `credentials: 'include'` in fetch calls
- ✅ All mini-apps use bridge protocol (postMessage)
- ✅ No direct API calls from iframe

**Conclusion:** Mini-apps are already following best practices. No changes needed.

### CORS Configuration Issue ⚠️ Identified

**Current Problem:** Gateway main.go uses:
```go
AllowCredentials: true  // ❌ Incompatible with Bearer tokens
```

**Why It's a Problem:**
- When `AllowCredentials: true`, cannot use multiple origins or wildcards
- Bearer tokens don't need credentials (not sent via cookies)
- Creates unnecessary attack surface
- Locks out production scaling

**Solution:** Use new CORS middleware with:
```go
AllowCredentials: false  // ✅ Correct for Bearer tokens
```
