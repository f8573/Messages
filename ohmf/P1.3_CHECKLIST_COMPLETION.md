```go
func (s *Service) TerminateSessionsForRelease(ctx context.Context, appID, version, reason string) (int64, error) {
  // Sends "release.suspended" event to each active session
  body := map[string]any{
    "app_id": appID,
    "version": version,
    "reason": reason,
    "event_at": time.Now().UTC(),
  }
  _, _ = s.AppendEvent(ctx, sessionID, "", "release.suspended", body)
  _ = s.EndSession(ctx, sessionID)
}
```

## ARCHITECTURE DIAGRAM

```
Admin Action
    |
    v
POST /v1/admin/apps/{app_id}/releases/{version}/suspend
    |
    +-> Apps Service
        |
        +-> Database: Set suspended_at, suspension_reason
        |
        +-> Audit Log: Record action
        |
        +-> Redis Pubsub: Publish to miniapp:release:invalidation
        |
        +-> Metrics: Log event to cache_invalidation_events
        |
        v
    Gateway Service (listening on pubsub)
    |
    +-> Receive invalidation event
    |
    +-> Query: SELECT id FROM miniapp_sessions WHERE app_id = ?
    |
    +-> For each session:
        |
        +-> AppendEvent: "release.suspended"
        |
        +-> EndSession: Set ended_at = now()
    |
    v
Active Session Receives Event
    |
    +-> Browser/App gets "release.suspended" event
    |
    +-> Can save state, show UI message
    |
    +-> Session automatically closed

Concurrent: New Session Creation
    |
    +-> POST /v1/miniapp/sessions (app_id)
    |
    +-> Gateway calls CheckReleaseStatus()
    |
    +-> Query: SELECT suspended_at, revoked_at...
    |
    +-> If suspended: Return HTTP 403
    |
    v
User Sees Error
    |
    +-> HTTP 403 Forbidden
    |
    +-> Error Code: "release_suspended"
    |
    +-> Message: "this app release has been suspended"
```

## OPERATIONAL PROCEDURES

### Suspend a Release (Admin)
```bash
curl -X POST http://apps-service:18086/v1/admin/apps/myapp/releases/1.0.0/suspend \
  -H "X-User-ID: admin-user-id" \
  -H "X-User-Role: admin" \
  -H "Content-Type: application/json" \
  -d '{"reason": "Security vulnerability: CVE-2026-1234"}'
```

**Result**:
- New sessions blocked immediately (HTTP 403)
- Active sessions receive "release.suspended" event
- Suspended sessions terminated within 1-2 seconds
- Audit log entry created
- Metrics recorded in cache_invalidation_events

### Reactivate a Release (Admin)
```bash
curl -X POST http://apps-service:18086/v1/admin/apps/myapp/releases/1.0.0/approve \
  -H "X-User-ID: admin-user-id" \
  -H "X-User-Role: admin"
```

**Result**:
- suspended_at = NULL
- New sessions allowed
- Users can create new sessions
- Old terminated sessions remain ended
